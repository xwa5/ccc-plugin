# 百度 Coding Config 助手

## 工具概述

`baidu-ccc-dev` 是一个用于管理和同步百度团队 Claude 配置的 CLI 工具。它可以帮助团队成员：
- 安装团队统一的 commands、skills、hooks、agents 配置
- 检测并解决本地与远程配置的冲突
- 查看当前配置状态和同步记录

## 何时使用

当用户提到以下场景时，主动使用此工具：
- "安装团队配置"
- "同步 Claude 配置"
- "更新我的 coding config"
- "查看配置状态"
- "配置是否是最新的"
- "团队有哪些共享的 commands/skills"

## 如何使用

### 1. 检查工具是否已安装

```bash
baidu-ccc-dev --version
```

如果未安装，指导用户安装：
```bash
npm install -g baidu-ccc-dev
```

### 2. 查看当前配置状态

```bash
baidu-ccc-dev status
```

这会显示：
- 配置目录位置
- 配置来源 URL
- 安装记录文件位置

### 3. 安装/更新配置

```bash
baidu-ccc-dev install
```

安装过程中会：
- 提示选择配置级别（个人级 `~/.claude/` 或项目级 `./.claude/`）
- 下载最新的团队配置
- 检测本地与远程的冲突
- 允许用户选择保留本地或使用远程版本
- 显示文件变更摘要（新增、更新、删除）

### 4. 自定义下载源（可选）

如果需要从自定义 URL 下载配置：
```bash
baidu-ccc-dev install <custom-url>
```

## 冲突处理策略

当检测到冲突时，工具会提供以下选项：
- **查看差异**：并排显示本地和远程版本的差异
- **保留本地版本**：跳过更新，保持当前本地文件
- **使用远程版本**：覆盖本地文件，使用团队最新配置

建议：
- 如果你修改过配置，选择 "查看差异" 了解变更内容
- 如果你的配置是自定义的，选择 "保留本地版本"
- 如果你想与团队保持一致，选择 "使用远程版本"

## 最佳实践

1. **定期同步**：建议每周运行一次 `baidu-ccc-dev install` 获取最新配置
2. **冲突处理**：遇到冲突时先查看差异，避免覆盖重要的自定义配置
3. **配置级别**：
   - 个人级 (`~/.claude/`)：跨项目共享的配置
   - 项目级 (`./.claude/`)：特定项目的配置
4. **脚本安装**：安装后检查 `~/.local/bin/` 是否在 PATH 中

## 示例对话

**用户**: "帮我安装团队的 Claude 配置"

**你的回应**：
1. 运行 `baidu-ccc-dev status` 查看当前状态
2. 运行 `baidu-ccc-dev install` 开始安装
3. 根据交互提示帮助用户选择配置级别和处理冲突
4. 安装完成后，总结变更内容（新增了哪些 commands/skills 等）

**用户**: "我的配置是最新的吗？"

**你的回应**：
1. 运行 `baidu-ccc-dev status` 查看同步记录
2. 建议运行 `baidu-ccc-dev install` 检查更新
3. 如果有更新，展示变更内容

## 注意事项

- 此工具不会自动推送本地配置到远程（单向同步）
- 如需贡献配置到团队，需要通过其他方式（如 Git PR）
- 删除的文件会自动清理，无需手动删除
- 空目录会自动清理
